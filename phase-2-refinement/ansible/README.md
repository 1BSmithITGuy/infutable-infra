## Purpose

Configures servers after Terraform provisions them. Playbooks are organized by a target OS under `playbooks/`; roles are flat under `roles/`.  

> NOTE:  Ansible host inventory files are generated by Terraform.

## Directory Structure

```
ansible/
    playbooks/windows/
        deploy-dc.yml           AD Domain Controller pipeline
    roles/
        initialize-disk/        Windows: Partition, format, create directories
        dc-promotion/           Windows: Install AD DS, promote to DC
        domain-join/            Windows: Join a server to the domain
    inventory/us103/
        groups.yml                  Group hierarchy (static)
        domain-controllers.yml      Site us103 AD DC inventory (Terraform managed)
        group_vars/
            all.yml                 Site us103 variables (DNS servers)
            domain_controllers.yml  AD DC specific variables
            windows.yml             WinRM connection settings
    scripts/
        deploy-dc.sh            AD Domain Controller pipeline wrapper (logging)
    ansible.cfg
```

Each new server type gets its own playbook.  


## Inventory

Inventory is organized per site in its own directory (`inventory/<site-code>/`). 

Within a site's inventory directory:

- `groups.yml` - Defines the static group hierarchy (windows, linux, etc.).
- Ansible host inventory is generated by Terraform from the deployment's tfvars. 
- `group_vars/all.yml` is site scoped; when additional sites are added, each gets its own `all.yml`.
- **Naming convention:** filenames use kebab-case *(domain-controllers.yml)*; group names and group_vars/ filenames use underscores *(domain_controllers.yml)*.  

## Roles

| Role | Platform | Description |
|------|-----|---------|
| `initialize-disk` | Windows | Initialize a raw disk (GPT), partition, format (NTFS), create directories. |
| `dc-promotion` | Windows | Install AD DS feature and promote to domain controller. |
| `domain-join` | Windows | Join a server to the AD domain. |

### Roles - required collections

```bash
ansible-galaxy collection install microsoft.ad community.windows
```

## Logs

Ansible's rolling log:  `/srv/logs/ansible/ansible.log`

Each playbook should have a wrapper script in `scripts/` that logs to `/srv/logs/ansible/<siteCode>/<hostname>/`.
